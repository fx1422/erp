## 🎯 测试用例设计亮点

### 1. 完整的测试场景覆盖

每个测试用例包含：
- ✅ 用例 ID 和名称
- ✅ 优先级（P0/P1）
- ✅ 前置条件
- ✅ 测试数据
- ✅ 详细测试步骤
- ✅ 预期结果
- ✅ 对应自动化脚本路径

### 2. 边界条件测试

包含大量边界条件测试：
- 用户名/编码重复校验
- 必填字段验证
- 库存不足拦截
- 已入库订单不可编辑
- 账号禁用登录拦截
- Token 过期验证

### 3. 流程完整性测试

设计了端到端流程测试场景：
- 采购流程：申请 → 审批 → 订单 → 入库
- 销售流程：报价 → 订单 → 出库
- 库存流程：入库 → 台账 → 出库 → 预警

## 🔧 自动化测试框架

### 技术栈

- **测试运行器**: Vitest
- **HTTP 测试**: Supertest
- **数据库**: Prisma Client
- **数据 Mock**: @faker-js/faker
- **密码加密**: bcrypt

### 测试架构

```
┌─────────────────────────────────────────┐
│           Vitest Test Runner            │
└─────────────────────────────────────────┘
                    │
        ┌───────────┼───────────┐
        ↓           ↓           ↓
   ┌────────┐  ┌────────┐  ┌────────┐
   │  Unit  │  │Integration│ │  E2E   │
   │ Tests  │  │  Tests   │ │ Tests  │
   └────────┘  └────────┘  └────────┘
        │           │           │
        └───────────┼───────────┘
                    ↓
        ┌─────────────────────┐
        │   Test Data Layer   │
        │  Factories / Seeds  │
        └─────────────────────┘
```

### 测试数据管理

- **种子数据**: SQL 脚本预置基础数据
- **数据工厂**: TypeScript Factory 模式创建测试数据
- **数据清理**: 测试后自动清理，避免污染

## 📝 使用说明

### 运行测试

```bash
# 进入项目目录
cd /home/admin/.openclaw/workspace/erp

# 安装依赖
npm install

# 设置测试数据库
export TEST_DATABASE_URL="postgresql://test:test@localhost:5432/erp_test"

# 运行所有测试
npm test

# 运行集成测试
npm run test:integration

# 生成覆盖率报告
npm run test:coverage
```

### 查看测试报告

```bash
# JUnit 报告
cat test/reports/junit-report.xml

# 覆盖率报告
open coverage/index.html
```

### 添加新测试

1. 在 `test/test-cases/` 添加测试用例文档
2. 在 `test/automation/` 添加自动化测试代码
3. 在 `test/data/` 添加测试数据（如需要）
4. 运行测试验证
