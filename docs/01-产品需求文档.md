# ERP 系统基础模块 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
企业资源规划 (ERP) 系统是企业管理的核心信息化平台，整合企业各部门业务流程，实现数据共享和业务流程自动化。

### 1.2 项目目标
- 构建可本地部署的 ERP 系统基础框架
- 实现核心业务模块的闭环管理
- 提供可扩展的架构设计，支持后续模块迭代
- 确保系统安全、稳定、易维护

### 1.3 目标用户
- 中小型企业
- 需要本地化部署的企业
- 对数据安全性要求较高的组织

## 3. 技术架构建议

### 3.1 技术选型

#### 前端
- **框架**: Vue 3 + TypeScript
- **UI 组件**: Element Plus
- **状态管理**: Pinia
- **构建工具**: Vite
- **HTTP 客户端**: Axios

#### 后端
- **语言**: Node.js / Java (Spring Boot) / Python (FastAPI)
- **推荐**: Node.js + NestJS (全栈 JS 统一)
- **API 规范**: RESTful API
- **认证**: JWT Token

#### 数据库
- **主数据库**: PostgreSQL / MySQL 8.0+
- **缓存**: Redis
- **ORM**: Prisma / TypeORM

### 3.2 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                      前端层 (Vue 3)                       │
│                    Element Plus UI                       │
└─────────────────────────────────────────────────────────┘
                          ↓ HTTPS
┌─────────────────────────────────────────────────────────┐
│                    API 网关层 (Nginx)                     │
│                  负载均衡 / 静态资源                       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   应用服务层 (NestJS)                     │
│            业务逻辑 / 权限验证 / 数据校验                   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                     数据访问层                            │
│         ORM (Prisma) / Redis 缓存 / 消息队列               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   数据存储层                              │
│          PostgreSQL / MySQL + Redis                      │
└─────────────────────────────────────────────────────────┘
```

## 4. 优先级划分

### MVP (第一版本 - 4-6 周)
- ✅ 系统管理：用户、角色、权限
- ✅ 基础数据：组织、供应商、客户、物料
- ✅ 采购管理：申请、订单、入库
- ✅ 销售管理：报价、订单、出库
- ✅ 库存管理：出入库、台账、预警

### 迭代 2 (2-3 周)
- 财务管理：应收应付
- 库存盘点、调拨
- 报表统计

### 迭代 3 (2-3 周)
- 生产管理（如需要）
- 工作流引擎
- 移动端适配

## 5. 部署方案 (本地部署)

### 5.1 部署环境要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2 核 | 4 核 + |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 50GB+ SSD |
| 操作系统 | Linux / Windows | Ubuntu 22.04 LTS |

### 5.2 部署方式

#### 方案 A: Docker Compose (推荐)
```yaml
services:
  frontend:
    image: erp-frontend:latest
    ports:
      - "80:80"
  
  backend:
    image: erp-backend:latest
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://...
      - REDIS_URL=redis://redis:6379
  
  database:
    image: postgres:15
    volumes:
      - pgdata:/var/lib/postgresql/data
  
  redis:
    image: redis:7-alpine
```

#### 方案 B: 传统部署
- Nginx 反向代理
- PM2 管理 Node.js 进程
- Systemd 管理数据库服务

### 5.3 备份策略
- 数据库每日自动备份
- 备份文件保留 7 天
- 支持手动备份和恢复
