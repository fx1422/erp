# ERP 项目快速启动指南

## 方式一：使用初始化脚本（推荐）

```bash
# 1. 进入项目目录
cd /home/admin/.openclaw/workspace/erp

# 2. 运行初始化脚本
bash scripts/init.sh

# 3. 启动开发环境
# 终端 1 - 启动后端
npm run dev:backend

# 终端 2 - 启动前端
npm run dev:frontend
```

## 方式二：手动启动

### 1. 启动数据库和缓存

```bash
# 使用 Docker Compose
docker-compose up -d postgres redis

# 或手动安装 PostgreSQL 和 Redis
```

### 2. 启动后端

```bash
cd backend

# 安装依赖
npm install

# 复制环境变量文件
cp .env.example .env

# 启动开发服务器
npm run start:dev
```

后端将运行在 http://localhost:3000

### 3. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端将运行在 http://localhost:5173

## 方式三：Docker Compose 一键启动

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

服务访问地址:
- 前端：http://localhost:80
- 后端 API: http://localhost:3000
- Nginx 代理：http://localhost:8080

## 常见问题

### 1. 端口被占用

修改 `docker-compose.yml` 中的端口映射或停止占用端口的服务。

### 2. 数据库连接失败

检查 PostgreSQL 是否正常运行：
```bash
docker-compose ps
docker-compose logs postgres
```

### 3. 前端无法访问后端

检查前端 `vite.config.ts` 中的代理配置是否正确。

### 4. 依赖安装失败

清除缓存后重试：
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```
