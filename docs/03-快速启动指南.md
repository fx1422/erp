## 🚀 启动方案

### 方案 A: 本地直接运行（推荐 ⭐）

无需 Docker，直接使用已安装的 node_modules 运行。

#### 前置条件

```bash
# 1. 确认 Node.js 版本
node --version  # 需要 v20+
npm --version

# 2. 确认 MySQL 已安装并运行
mysql --version
# 或者使用 Docker 只启动数据库
docker run -d --name erp-mysql \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=erp_db \
  -p 3306:3306 \
  mysql:8.0

# 3. 确认 Redis 已安装并运行
redis-cli ping  # 应返回 PONG
# 或者使用 Docker
docker run -d --name erp-redis \
  -p 6379:6379 \
  redis:7-alpine
```

#### 启动步骤

```bash
cd /home/admin/.openclaw/workspace/erp

# 1. 启动后端
cd backend
npm run start:dev
# 后端将在 http://localhost:3000 运行

# 新开终端

# 2. 启动前端
cd frontend
npm run dev
# 前端将在 http://localhost:5173 运行
```

#### 一键启动脚本

```bash
./scripts/start-dev.sh
```

---

### 方案 B: 使用国内镜像源

修改 `docker-compose.yml`，使用国内镜像加速器：

```yaml
services:
  mysql:
    image: registry.cn-hangzhou.aliyuncs.com/library/mysql:8.0
  
  redis:
    image: registry.cn-hangzhou.aliyuncs.com/library/redis:7-alpine
  
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
  
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
```

然后启动：

```bash
cd /home/admin/.openclaw/workspace/erp
sudo docker-compose up -d --build
```

---

### 方案 C: 手动下载镜像后导入

在有网络的机器上下载镜像，然后导出导入：

```bash
# 在有网络的机器上
docker pull mysql:8.0
docker pull redis:7-alpine
docker save mysql:8.0 > mysql.tar
docker save redis:7-alpine > redis.tar

# 传输到目标机器
scp mysql.tar redis.tar user@target:/tmp/

# 导入镜像
docker load < /tmp/mysql.tar
docker load < /tmp/redis.tar

# 启动
sudo docker-compose up -d
```

## 🐛 常见问题

### 问题 1: 后端启动失败 - 数据库连接错误

**症状**:
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```

**解决**:
```bash
# 检查 MySQL 是否运行
mysqladmin ping -uroot -p

# 查看后端日志
cd backend
npm run start:dev 2>&1 | grep -i error
```

### 问题 2: 前端启动失败 - 端口被占用

**症状**:
```
Error: Port 5173 is already in use
```

**解决**:
```bash
# 查找占用端口的进程
lsof -i :5173

# 杀死进程
kill -9 <PID>

# 或者修改 vite.config.ts 使用其他端口
```

### 问题 3: 后端启动失败 - 依赖未安装

**症状**:
```
Error: Cannot find module '@nestjs/core'
```

**解决**:
```bash
cd backend
npm install
```

### 问题 4: 前端启动失败 - 依赖未安装

**症状**:
```
Error: Cannot find module 'vue'
```

**解决**:
```bash
cd frontend
npm install
```

## 🔧 开发环境配置

### 后端环境变量

编辑 `backend/.env`:

```env
NODE_ENV=development
PORT=3000
FRONTEND_URL=http://localhost:5173

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=root
DB_DATABASE=erp_db

# JWT 配置
JWT_SECRET=erp-secret-key-change-in-production

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379
```

### 前端环境变量

编辑 `frontend/.env` (如需要):

```env
VITE_API_BASE_URL=http://localhost:3000/api
VITE_APP_TITLE=ERP 系统
```
